Button Component

A versatile button component that supports multiple types, sizes, states, and can include icons. The button can be rendered as either a button or anchor element based on the presence of an href prop.

Properties

Property Type Default Description
label String - The label text of the button
type String secondary Type of the Button component. Can be "primary", "secondary", "danger" or "link"
isSuccess Boolean false If true, shows button in success color. With primary type shows filled success button, with secondary type shows outlined success button.
size String md Size of the button. Can be "sm", "md" or "lg"
href String - Required for link buttons. Renders as <a> tag when provided, else <button>
icon React Element - Icon from orion-icons library. Pass inheritFontColor prop to icon for hover color changes
disabled Boolean false Disables the button when true
active Boolean false Makes button appear in active state
block Boolean false Makes button full width
onClick Function - Click handler function
onMouseDown Function - Mouse down handler function
onMouseUp Function - Mouse up handler function
className String - Custom CSS class
dataTestId String orion-button Custom data test ID for automation

Usage Examples

Basic Button Types

import Button from "orion/lib/button";

<Button label="Primary Button" type="primary" />
<Button label="Secondary Button" />
<Button label="Danger Button" type="danger" />
<Button label="Link Type" type="link" href="#test" />

Button with Icon

import Button from "orion/lib/button";
import Filter from "@druva-apps/orion-icons/lib/FilterGreyOutline";

<Button icon={<Filter inheritFontColor />} />
<Button
  icon={<Filter inheritFontColor inheritFontSize />}
  label="Filter"
  type="primary"
/>

Best Practices

  • Use primary buttons for main actions
  • Use secondary buttons for alternative actions
  • Use danger buttons for destructive actions
  • Use link buttons for navigation or less important actions
  • When using icons, ensure they are from the orion-icons library
  • Keep button labels clear and concise
  • Use appropriate button sizes based on their importance and context
