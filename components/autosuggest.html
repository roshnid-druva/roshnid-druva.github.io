AutoSuggest Component

A component that provides suggestions as the user types in an input field, with support for grouping, highlighting matches, and custom rendering.

Import

import AutoSuggest from "orion/lib/autoSuggest";

Properties

Property Type Default Description
data Array - Array of options for suggestions. (Required)
defaultValue String - Default value for the input field
placeholder String - Placeholder text for the input
className String - Custom CSS class for the container
renderMenuItem Function - Custom renderer for menu items
getItemValue Function - Function to determine the input value after selection
filterItem Function - Custom filtering function for suggestions
onFocus Function - Called when input receives focus (event, value)
onChange Function - Called when input value changes (value)
onClear Function - Called when the input is cleared
onBlur Function - Called when input loses focus (event, value)
onEnter Function - Called when Enter key is pressed (value)
onItemSelect Function - Called when an item is selected (selectedItem)
disabled Boolean false Disables the input field
showEmptyMessage Boolean false Shows a message when no matching results are found
maxResultsLimit Number 10 Maximum number of results to display
minChars Number 1 Minimum characters required before showing suggestions
highlightMatches Boolean false Highlights matching text in results
height Number 300 Height of the results container in pixels
async Boolean false For asynchronous data loading
dataTestId String "orion-autosuggest" Custom data test ID for automation

Data Format

The data prop accepts the following formats:

Simple Options

[{ label: "Linux" }, { label: "Microsoft Windows" }, { label: "SQL Server" }];

Grouped Options

[
  {
    label: "Operating Systems",
    items: [{ label: "Linux" }, { label: "Microsoft Windows" }],
  },
  {
    label: "Databases",
    items: [{ label: "SQL Server" }, { label: "Oracle" }],
  },
];

Mixed Options (both grouped and non-grouped)

[
  {
    label: "Operating Systems",
    items: [{ label: "Linux" }, { label: "Microsoft Windows" }],
  },
  { label: "SQL Server" },
  { label: "Oracle" },
];

Usage Examples

Basic Usage with Simple Options

const data = [
  { label: "Linux" },
  { label: "Microsoft Windows" },
  { label: "SQL Server" },
  { label: "Oracle" },
];

<AutoSuggest data={data} showEmptyMessage />;

With Grouped Options

const data = [
  {
    label: "Druva Phoenix Data Sources",
    items: [
      { label: "Linux" },
      { label: "Microsoft Windows" },
      { label: "SQL Server" },
    ],
  },
  {
    label: "Druva InSync Data Sources",
    items: [
      { label: "Salesforce" },
      { label: "Office365" },
      { label: "GSuite" },
    ],
  },
];

<AutoSuggest data={data} />;

With Default Value

<AutoSuggest defaultValue="Microsoft Windows" data={data} />

Asynchronous Data Loading

<AutoSuggest
  async
  data={asyncData}
  onChange={(value) => {
    // Fetch data based on value
    setTimeout(() => {
      setAsyncData(fetchedData);
    }, 500);
  }}
  onItemSelect={() => setAsyncData([])}
  onBlur={() => setAsyncData([])}
/>

Highlighting Matches

<AutoSuggest data={data} highlightMatches />

Custom Menu Item Renderer

const data = [
  { label: "Linux", percentage: 30 },
  { label: "Microsoft Windows", percentage: 60 },
  { label: "SQL Server", percentage: 50 },
];

<AutoSuggest
  data={data}
  renderMenuItem={(item, renderHighlightedText) => (
    <div
      style={{
        display: "flex",
        justifyContent: "space-between",
        alignItems: "center",
      }}
    >
      <span>{renderHighlightedText(item.label)}</span>
      <span>{`${item.percentage}%`}</span>
    </div>
  )}
/>;

Behavior Notes

  • The component filters options as the user types, matching against the label property by default
  • Grouping is supported by adding an items array to an option
  • When async is true, filtering must be handled externally through the onChange handler
  • The renderHighlightedText function passed to renderMenuItem handles highlighting matched text
  • When highlightMatches is true, matching portions of text are emphasized
  • The onEnter handler is called when Enter is pressed with no selection; if an item is highlighted, onItemSelect is called instead
  • Items with matching text are shown until the maxResultsLimit is reached
