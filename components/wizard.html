Wizard Component

A step-by-step interface component for guiding users through complex processes.

Import

import Wizard from "orion/lib/wizard";

Properties

Property Type Default Description
items Array - Array of step items for the wizard. (Required)
currentStepIndex Number 0 Index of the current active step. (Required)
toggle Function - Function to toggle wizard visibility. (Required)
header String - Header text for the wizard. (Required)
isOpen Boolean false Controls whether the wizard is displayed.
size String "xxl" Size of the wizard modal: "sm", "md", "lg", "xl", or "xxl".
closeOnEscape Boolean true Whether to close the wizard when the escape key is pressed.
onPrevClick Function - Callback for when the previous button is clicked.
onNextClick Function - Callback for when the next button is clicked.
onFinishClick Function - Callback for when the finish button is clicked.
onOpened Function - Callback when the wizard is fully opened.
onClosed Function - Callback when the wizard is fully closed.
docUrl String - URL for help documentation (displays an icon in the header).
docUrlTooltipText String - Tooltip text for the documentation link icon.
alert Object - Wizard-level alert/notification configuration.
className String - Custom CSS class for the wizard container.
disableFinish Boolean false Disables the finish button.
disableNext Boolean false Disables the next button.
disablePrev Boolean false Disables the previous button.
bodyClassName String - Custom CSS class for the wizard modal body.
bodyHeight String/Number - Height of the wizard body in pixels (constrained by viewport height).
wizardBodyClassName String - Custom CSS class for the steps container.
prevLabel String "Previous" Custom text for the previous button.
nextLabel String "Next" Custom text for the next button.
finishLabel String "Finish" Custom text for the finish button.
dataTestId String "orion-wizard" Custom data test ID for automation.

Item Object Structure

Each item in the items array should have the following structure: 

{
  title: String,       // Title displayed in the wizard steps
  component: Function  // Function that returns the component to render for this step
}

Alert Object Structure

The alert prop accepts an object with the following structure: 

{
  show: Boolean,                    // Whether to display the alert
  closable: Boolean,                // Whether the alert can be closed
  onClose: Function,                // Callback when alert is closed
  message: String | React.Element,  // Alert content
  type: String                      // Alert type: "success", "warning", "danger", or "info"
}

constructor(props) {
  super(props);
  this.state = {
    currentStepIndex: 0,
    isWizardOpen: false,
    alert: {
      show: false,
      message: ""
    }
  };

  this.items = [
    {
      title: "Plan Details",
      component: () => <PlanDetailsComponent />
    },
    {
      title: "VM Selection",
      component: () => <VMSelectionComponent />
    },
    {
      title: "Failover Settings",
      component: () => <FailoverSettingsComponent />
    }
  ];
}

toggleWizard = () => {
  this.setState({
    isWizardOpen: !this.state.isWizardOpen
  });
};

handlePrevClick = () => {
  this.setState({
    currentStepIndex: this.state.currentStepIndex - 1
  });
};

handleNextClick = () => {
  this.setState({
    currentStepIndex: this.state.currentStepIndex + 1
  });
};

handleFinishClick = () => {
  // Handle finish action
  this.toggleWizard();
};

render() {
  return (
    <>
      <Button
        label="Launch Wizard"
        type="primary"
        onClick={this.toggleWizard}
      />
      <Wizard
        isOpen={this.state.isWizardOpen}
        toggle={this.toggleWizard}
        header="Configuration Wizard"
        items={this.items}
        currentStepIndex={this.state.currentStepIndex}
        onPrevClick={this.handlePrevClick}
        onNextClick={this.handleNextClick}
        onFinishClick={this.handleFinishClick}
      />
    </>
  );
}

Behavior Notes

  • The wizard uses a modal interface with navigation between steps.
  • The currentStepIndex controls which step is currently displayed.
  • You must handle the step navigation logic in your component by updating the currentStepIndex prop.
  • The final step shows a "Finish" button instead of "Next".
  • For smaller wizards (size "sm"), some steps might not be displayed in the header.
  • The bodyHeight prop allows controlling the content area height, but is constrained by the viewport height.
