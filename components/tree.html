Tree Component

A hierarchical component for displaying and selecting nested tree data, with support for various customization options.

Import

import Tree from "orion/lib/tree";

Properties

Property Type Default Description
nodes Array - Array of node objects to render in the tree. (Required)
checked Array [] Array of unique node IDs that should be selected.
expanded Array [] Array of unique node IDs that should be expanded.
loading Array [] Array of unique node IDs that should show a loading spinner.
disabled Boolean false Disables the entire tree.
checkable Boolean true Whether nodes can be selected via checkboxes.
showNodeIcon Boolean true Whether to show icons for nodes.
expandDisabled Boolean false Prevents nodes from being expanded.
showHiddenNodes Boolean false Controls visibility of hidden nodes.
uniqueDataKey String "value" Property name for the unique identifier of each node.
hiddenProp String "hidden" Property name that identifies hidden nodes.
disabledProp String "disabled" Property name that identifies disabled nodes.
labelProp String "label" Property name for the node display text.
tooltipProp String "tooltipText" Property name for the node tooltip text.
checkableProp String "checkable" Property name that identifies if a node can be selected.
childrenProp String "children" Property name for a node's child nodes.
childrenLevelMap Object null Map of level to children property name for non-consistent data structures.
isStaticData Boolean false Optimizes for static trees without dynamic children loading.
heading String - Optional heading text for the tree.
returnOnlyLeafNodes Boolean false Only return leaf nodes in the checked array.
onCheck Function - Callback when node selection changes: (checked, node, nodes) => void
onExpand Function - Callback when node expansion changes: (expanded, loading, nodeState, node) => void
renderIcon Function - Custom renderer for node icons: (node) => React.Element
renderLabel Function - Custom renderer for node labels: (node) => React.Element
renderSubLabel Function - Custom renderer for secondary node labels: (node) => React.Element
renderNodeAction Function - Custom renderer for right-side node actions: (node) => React.Element
getCheckboxTooltip Function - Function to get tooltip for node checkboxes: (node) => String or Object
noCascade Boolean false When true, parent and child selection states are independent.
optimisticToggle Boolean false Special selection behavior for restore use cases.
allowSelectAll Boolean false Adds a select all checkbox to the header.
dataTestId String "orion-tree" Custom data test ID for automation.

Node Object Structure

Each node in the nodes array should have the following structure: 

{
  label: String,              // Display text for the node
  value: String,              // Unique identifier for the node
  tooltipText: String,        // Optional tooltip text
  children: Array,            // Child nodes
  isLeaf: Boolean,            // Whether the node is a leaf node
  isHidden: Boolean,          // Whether the node should be hidden
  isDisabled: Boolean,        // Whether the node should be disabled
  type: String,               // Optional type information (useful for icons)
  renderComponent: Function,  // Optional function to render custom node content
  className: String           // Optional CSS class for the node
}

onNodeCheck = (checked, node, nodes) => {
  this.setState({
    checked
  });
};

onNodeExpand = (expanded, loading, nodeState, node) => {
  this.setState({
    expanded
  });
};

render() {
  return (
    <div style={{ height: "300px" }}>
      <Tree
        nodes={this.treeNodes}
        expanded={this.state.expanded}
        onExpand={this.onNodeExpand}
        checked={this.state.checked}
        onCheck={this.onNodeCheck}
      />
    </div>
  );
}

renderIcon = (node) => {
  switch (node.type) {
    case "ppt":
      return <Icon src={PptIcon} />;
    case "doc":
      return <Icon src={DocIcon} />;
    case "img":
      return <Icon src={ImageIcon} />;
    default:
      return <Icon src={FolderIcon} />;
  }
};

renderNodeAction = (node) => {
  // Show file size for nodes that have it
  return typeof node.size === "number"
    ? `${parseFloat(node.size / 1024).toFixed(2)} KB`
    : null;
};

renderSubLabel = (node) => {
  return node.type ? `Type: ${node.type}` : null;
};

<Tree
  nodes={this.treeNodes}
  expanded={this.state.expanded}
  onExpand={this.onNodeExpand}
  checked={this.state.checked}
  onCheck={this.onNodeCheck}
  renderIcon={this.renderIcon}
  renderNodeAction={this.renderNodeAction}
  renderSubLabel={this.renderSubLabel}
/>;

Custom Node Labels

renderLabel = (node) => {
  if (node.isLink) {
    return (
      <a href={`/${node.label}`} onClick={(e) => e.preventDefault()}>
        {node.label}
      </a>
    );
  } else if (node.isButton) {
    return <Button type="secondary" label={node.label} size="sm" />;
  }
  return node.label;
};

<Tree
  nodes={this.treeNodes}
  expanded={this.state.expanded}
  onExpand={this.onNodeExpand}
  checked={this.state.checked}
  onCheck={this.onNodeCheck}
  renderLabel={this.renderLabel}
/>;

Tree with Loading States

onLoadingNodeExpand = (expanded, loading, node) => {
  if (!node.expanded) {
    this.setState({ expanded });
  } else {
    this.setState({ loading }, () => {
      // Simulate API call to fetch children
      setTimeout(() => {
        const nodeValue = node.value;
        const index = this.state.loading.indexOf(nodeValue);
        const newLoading = [...this.state.loading];
        newLoading.splice(index, 1);
        this.setState({
          expanded,
          loading: newLoading,
        });
      }, 1000);
    });
  }
};

<Tree
  nodes={this.treeNodes}
  expanded={this.state.expanded}
  onExpand={this.onLoadingNodeExpand}
  checked={this.state.checked}
  onCheck={this.onNodeCheck}
  loading={this.state.loading}
/>;

Behavior Notes

  • When optimisticToggle is true, only explicitly checked nodes are included in the checked array
  • The childrenLevelMap is useful for complex data structures with different property names for children at each level
  • You can create custom pagination by using the renderComponent and adding "Load More" nodes at appropriate positions
  • The getCheckboxTooltip function can return a string for a basic tooltip or an object that will be spread to the Tooltip component
  • When noCascade is false (default), checking a parent checks all children, and checking all children checks the parent
  • The tree component supports dynamically loading children by managing the loading state and modifying the node data
