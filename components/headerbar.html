Headerbar Component

A flexible, configurable header component that provides navigation, branding, and user profile functionality for applications.

Import

import Headerbar from "orion/lib/headerbar";

Properties

Property Type Default Description
brand Object - Configuration for brand display in the header
left Array [] Array of component objects to render on the left side
right Array [] Array of component objects to render on the right side
fixed Boolean true Whether the header should be fixed at the top of the page
consoleConfig Object - Configuration for console link (typically Druva console)
onGlobalMenuClick Function - Handler for global menu icon click
height String "56px" Height of the headerbar
dataTestId String "orion-headerbar" Custom data test ID for automation

Brand Object Properties

Property Type Description
name String Name of the product/service
productName String Name of the parent product (optional)
isActive Boolean Whether the brand link is active
href String URL for the brand link
logoUrl String URL to the logo image
renderComponent Function Custom renderer for brand content

Component Types

The Headerbar accepts various component types that can be placed in the left or right arrays:

Link Component

Druva
Users 
{
  component: "link",
  data: {
    name: "Users",
    href: "/users",
    isActive: true,
    onClick: (e) => {
      // Handle click
    }
  }
}

Dropdown Component

Druva

{

    ]
  }
}

Other Components

Additional component types include:

  • divider - Renders a vertical divider
  • button - Renders a button
  • select - Renders a select dropdown
  • icon-dropdown - Icon with dropdown menu
  • icon-popover - Icon with popover content
  • megamenu - Dropdown with multiple columns of links
  • custom - For rendering custom components

Behavior Notes

  • The Headerbar provides a flexible container for application navigation and branding
  • By default, the Headerbar is fixed to the top of the page
  • The component can be configured with various types of navigation elements
  • Brand information appears on the left side, while profile and action elements typically appear on the right
  • Multiple component types can be combined to create a custom header experience
  • The height of the Headerbar is customizable to match the application design
