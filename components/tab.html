Tab Component

A component for organizing content into tabs that can be selected to display corresponding content.

Import

import Tab from "orion/lib/tab";

Properties

Property Type Default Description
items Array - List of tab detail objects. (Required)
activeTabId String - Tab ID for the active tab.
tabbar Boolean false If true, only tab triggers will be rendered without the content components.
onTabClick Function - Callback when a new tab is clicked.
className String - Custom class for the component.
variant String "default" Style variant: "default", "classic", or "widget".
borderBottom Boolean false Adds border bottom to the tabbar header.
dataTestId String "orion-tabs" Custom data test ID for automation.

Tab Item Object

Each object in the items array should have the following structure: 

{
  id: string,          // Unique identifier for the tab
  title: string,       // Title of the tab
  isInvalid: boolean,  // Flag to indicate invalid content
  component: function, // Function that returns content to render
  renderComponent: function // Function to create custom tab links
}
  • id: Unique identifier for the tab
  • title: Title displayed in the tab
  • isInvalid: Boolean flag indicating the tab contains invalid form fields
  • component: Callback function that returns the component to be rendered in the tab container
  • renderComponent: Use this to create links using custom components or router libraries like react-router

this.state = {
  activeTabId: "tab_id_1"
}

onTabClick = (newTab) => {
  this.setState({
    activeTabId: newTab.id
  })
}

const tabItems = [
  {
    title: "Tab 1",
    id: "tab_id_1",
    component: () => (
      <div>
        <h6>Tab Content 1 with some lorem ipsum text</h6>
        <div>Lorem ipsum dolor sit amet, consectetur adipiscing elit...</div>
      </div>
    )
  },
  {
    title: "Tab 2",
    id: "tab_id_2",
    isInvalid: true,
    component: () => (
      <div>
        <h6>Tab Content 2 with some lorem ipsum text</h6>
        <div>Pellentesque lobortis at justo at consequat...</div>
        <br />
        This tab contains some invalid content.
      </div>
    )
  },
  {
    title: "Tab 3",
    id: "tab_id_3",
    component: () => (
      <div>
        <h6>Tab Content 3 with some lorem ipsum text</h6>
        <div>Duis tortor odio, auctor vitae rhoncus id...</div>
      </div>
    )
  }
]

<Tab
  items={tabItems}
  variant="default"
  activeTabId={this.state.activeTabId}
  onTabClick={this.onTabClick}
/>

const tabBarItems = [
  { title: "Tab 1", id: "tab_id_1" },
  { title: "Tab 2", id: "tab_id_2" },
  { title: "Tab 3", id: "tab_id_3" },
];

onTabBarClick = (newTab) => {
  this.setState({
    activeTabBarId: newTab.id,
  });
};

<Tab
  tabbar
  items={tabBarItems}
  activeTabId={this.state.activeTabBarId}
  onTabClick={this.onTabBarClick}
/>;

Behavior Notes

  • The Tab component does not provide any styling (not even padding) to the Tab content area, except for the "classic" variant.
  • When using renderComponent, make sure to use the provided otherClasses and activeClass parameters to maintain proper styling.
  • The isInvalid property can be used to indicate that a tab contains invalid form fields.
  • The Tab component requires controlled state management. You must provide the activeTabId and handle tab switching through the onTabClick callback.
  • Tab switching is the responsibility of the parent component. The Tab component does not handle the tab switching internally.
  • When tabbar is true, only the tab navigation is rendered, and content components are ignored.
