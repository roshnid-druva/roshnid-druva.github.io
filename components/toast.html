Toast Component

A component that displays temporary notifications as toast messages.

Import

import Toast from "orion/lib/toast";

Properties

Property Type Default Description
alerts Array - List of alerts to be displayed as toasts. (Required)
delay Number 5000 Time in milliseconds after which the toast message automatically closes.
onClose Function - Callback function when a toast is closed. (Required)
className String - Custom CSS class.
dataTestId String "orion-toast" Custom data test ID for automation.

Alert Object

Each alert in the alerts array should have the following structure: 

{
  message: String | React Element, // Content to display in the toast
  type: String                     // "success", "warning", or "danger"
}
  • message: The content to be displayed in the toast (can be text or JSX)
  • type: The visual style of the toast ("success", "warning", or "danger")

Usage Examples

Basic Usage

constructor(props) {
  super(props);
  this.state = {
    alerts: [],
    delay: 5000
  };
}

onCreateToast = (type) => {
  this.setState({
    alerts: [
      ...this.state.alerts,
      {
        type: type,
        message: (
          <div>
            This is {type} toast. <a href="#">Click Here</a> to check status.
          </div>
        )
      }
    ]
  });
};

closeAlert = (alert) => {
  let alerts = this.state.alerts.filter(alt => alert !== alt);
  this.setState({
    alerts: alerts
  });
};

render() {
  return (
    <div>
      <Toast
        alerts={this.state.alerts}
        delay={this.state.delay}
        onClose={this.closeAlert}
      />

      <button
        className="oui-btn oui-btn-success"
        onClick={() => this.onCreateToast("success")}
      >
        Success Toast
      </button>

      <button
        className="oui-btn oui-btn-warning"
        onClick={() => this.onCreateToast("warning")}
      >
        Warning Toast
      </button>

      <button
        className="oui-btn oui-btn-danger"
        onClick={() => this.onCreateToast("danger")}
      >
        Danger Toast
      </button>
    </div>
  );
}

With Custom Delay

<Toast
  alerts={this.state.alerts}
  delay={10000} // 10 seconds
  onClose={this.closeAlert}
/>

With ID for Tracking

It's recommended to include a unique identifier in your alert objects to help with managing them: 

onCreateToast = (type) => {
  this.setState({
    alerts: [
      ...this.state.alerts,
      {
        id: Date.now(), // unique identifier
        type: type,
        message: `This is a ${type} toast message`,
      },
    ],
  });
};

closeAlert = (alert) => {
  let alerts = this.state.alerts.filter((alt) => alt.id !== alert.id);
  this.setState({
    alerts: alerts,
  });
};

Behavior Notes

  • Toasts will automatically disappear after the specified delay (default: 5000ms)
  • Users can manually close toasts by clicking the (x) button
  • The onClose callback is triggered in both cases (timeout or manual close)
  • You must handle removing the alert from your state in the onClose function
  • It's recommended to add a unique identifier to each alert object to make it easier to manage them
  • Toasts are displayed in the top-right corner of the screen by default
  • New toast messages appear with an animation sliding in from the right
  • Multiple toasts stack vertically with the most recent at the top
