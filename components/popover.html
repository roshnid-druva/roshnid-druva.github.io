Popover Component

A component that displays content in a popup when a trigger element is clicked.

Import

import Popover from "orion/lib/popover";

Properties

Property Type Default Description
title String - Popover title.
body String/Function/Element - The body section of the Popover component. (Required)
type String "controlled" Type of popover: "controlled" or "uncontrolled".
isOpen Boolean false Toggles the popover visibility (for controlled popovers only). (Required for controlled type)
toggle Function - Function called when trigger is clicked (for controlled popovers only). (Required for controlled type)
hideOnOutsideClick Boolean - For controlled popovers, hides when clicked outside the popover.
placement String "auto" Position of the popover relative to the trigger.
disabled Boolean false Disables the popover functionality (not the trigger).
size String "sm" Width of the popover: "auto", "xs", "sm", "md", or "lg".
bodyClassName String - Custom class for the popover body.
trigger String "click" Event that triggers the popover, e.g., "click hover".
className String - Custom class for the popover container.

Types of Popovers

Controlled Popover

  • Requires managing the open/close state in your component
  • Used for interactive popovers (e.g., forms, filters)
  • Does not dismiss on blur by default

Uncontrolled Popover

  • Manages its own state internally
  • Used for displaying information
  • Automatically dismisses when clicked outside

Placement Options

Popover supports multiple placement options:

auto auto-start auto-end top top-start top-end right right-start right-end bottom bottom-start bottom-end left left-start left-end

Usage Examples

Controlled Popover

Used for interactive content like forms or filters.

Filters
constructor(props) {
  super(props);
  this.state = {
    isOpen: false
  };
}

toggle = () => {
  this.setState({
    isOpen: !this.state.isOpen
  });
};

render() {
  return (
    <Popover
      title="Filters"
      body={() => {
        return (
          <Form
            onSubmit={e => {
              e.preventDefault();
              this.toggle();
            }}
          >
            <FormGroup row>
              <Label for="policy" sm={4} text="Policy" />
              <Col sm={8}>
                <Input type="text" name="policy" id="policy" />
              </Col>
            </FormGroup>
            <FormGroup footer className="oui-mb-0 oui-text-right">
              <Input
                type="button"
                label="Cancel"
                onClick={this.toggle}
              />
              <Input type="submit" label="Apply" />
            </FormGroup>
          </Form>
        );
      }}
      isOpen={this.state.isOpen}
      placement="bottom"
      size="sm"
      toggle={this.toggle}
    >
      <Button
        icon={{
          classes: "di di-filter"
        }}
      />
    </Popover>
  );
}

Uncontrolled Popover

Used for displaying information. Automatically manages its own state.

<Popover
  type="uncontrolled"
  title="More Info"
  body={() => {
    return (
      <div>
        <Row className="oui-mb-3">
          <Col sm={4}>Site Title</Col>
          <Col sm={8}>Mohit_PubO365Groups</Col>
        </Row>
        <Row className="oui-mb-3">
          <Col sm={4}>Site URL</Col>
          <Col sm={8}>
            <a
              rel="noopener noreferrer"
              href="https://example.com"
              target="_blank"
            >
              https://example.com
            </a>
          </Col>
        </Row>
        <Row className="oui-mb-3">
          <Col sm={4}>Status</Col>
          <Col sm={8}>Enabled</Col>
        </Row>
        <Row className="oui-mb-3">
          <Col sm={4}>Added on</Col>
          <Col sm={8}>Nov 12 2018, 07:30</Col>
        </Row>
      </div>

  }}
  placement="right"
  size="md"
>
  <Icon classes="di di-2 di-information" actionable />
</Popover>

Behavior Notes

  • For form inputs inside popovers, use the controlled type to manage form submission
  • The body prop can accept a function that returns JSX, allowing dynamic content
  • When using controlled popovers with forms, remember to handle the form submission and toggle the popover accordingly
  • The trigger element is provided as a child to the Popover component
  • For controlled popovers, you must provide the isOpen state and the toggle function
  • Uncontrolled popovers are simpler to use as they manage their own state
  • The placement option automatically adjusts if there's not enough space in the specified direction
  • The popover can be triggered by different events by setting the trigger prop (e.g., "click", "hover")
