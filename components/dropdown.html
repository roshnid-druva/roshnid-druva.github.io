Dropdown Component

A component that displays a list of options in a pop-up menu when triggered.

Import

import Dropdown from "orion/lib/dropdown";

Properties

Property Type Default Description
items Array - Items to be displayed in the dropdown menu. (Required)
label String - Label for the dropdown trigger button.
icon React Element - Icon element from orion-icons library to be displayed on the trigger button.
type String "primary" Dropdown style type: "primary", "secondary", "tertiary", "icon", or "link".
split Boolean false Creates a split dropdown using button group. Not applied for "tertiary" and "link" types.
size String "md" Size of the dropdown: "sm", "md", or "lg". Not applicable for icon type.
disabled Boolean false Disables the entire dropdown.
disabledMain Boolean false Disables only the main button of the split dropdown.
disabledTrigger Boolean false Disables the trigger button of the dropdown.
maxMenuHeight String | Number 192 Maximum height of the dropdown menu.
rightAligned Boolean false Aligns the dropdown menu to the right of the trigger button.
onClick Function - Handler for the main button click in a split dropdown.
onItemClick Function - Handler for dropdown item clicks. The clicked item object is passed as argument.
className String - Provision to pass custom class.
dataTestId String "orion-dropdown" Provision to pass a custom data test ID for automation purpose.
direction "up" | "down" "down" Direction in which the dropdown menu opens.
enableLongPress Boolean false Enables long press behavior (500ms) on trigger button.

Items Array Structure

The items prop can accept an array with the following structure types:

Simple Item

{
  label: String | Function, // Text or render function for the item
  disabled: Boolean         // Whether the item is disabled or not
}

Group of Items

{
  heading: String,        // Group header text
  items: [                // Array of items in the group
    {
      label: String | Function,
      disabled: Boolean
    }
  ]
}

Divider Item

{
  divider: true; // Renders a separator line
}

Multi-Level Item

{
  label: String | Function,
  items: [                // Nested dropdown items
    {
      label: String | Function,
      disabled: Boolean,
      items: [...]        // Can be nested further
    }
  ]
}

Note: For multi-level dropdown menus, the overflow will always be visible and maxMenuHeight won't be applied if there is at least one item containing further children.

Behavior Notes

  • Clicking on the dropdown trigger button opens the dropdown menu
  • Clicking anywhere outside the dropdown menu closes it
  • For split dropdowns, the main button can perform an action while the arrow button opens the dropdown menu
  • When enableLongPress is set to true, holding the trigger button for 500ms will open the dropdown
  • Multi-level dropdowns appear as cascading menus
  • The dropdown menu position adjusts automatically if there is not enough space in the specified direction
  • Disabled menu items cannot be clicked but still appear in the dropdown
  • The component handles keyboard navigation for accessibility
